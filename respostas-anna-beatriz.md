1. Diferença entre git init e git clone?
Git init cria um repositório novo, git clone copia um repositório existente.

2. O que faz git status?
Mostra as alterações dos arquivos.

3. Para que serve git add antes do commit?
Para avaliar quais mudanças vão entrar no commit.

4. Diferença entre git pull e git fetch?
Fetch baixa mudanças sem aplicar, pull baixa e já aplica no código.

5. O que é branch e por que usar?
É uma cópia do repositório usada para organizar e separar tarefas.

6. O que é Pull Request?
É um pedido para juntar mudanças de uma branch em outra no GitHub.

7. Diferença entre branch de origem e destino em PR?
Origem é onde as mudanças estão e destino é onde elas serão aplicadas.

8. O que acontece se duas pessoas editam a mesma linha?
O Git gera um conflito que precisa ser resolvido.

9. Para que serve .gitignore?
Define arquivos que o Git deve ignorar.

10. Função do README.md?
Explica o projeto e como usar.
